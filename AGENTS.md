# Documentation starter instructions

## About this project

- This is a reusable documentation starter built as a Next.js app.
- The audience is anyone who wants to launch a docs site from a template.
- Pages are MDX files with YAML frontmatter.
- Site configuration lives in `site.config.ts`.
- Run `npm run dev` to preview locally.
- Run `npm run type-check` and `npm run build` before publishing.
- The template is designed for easy deployment on Vercel's free Hobby plan.

## Terminology

- Use `Next.js Docs Starter` for the default template name.
- Use `site.config.ts` for the main customization file.
- Use `docs site`, `documentation site`, or `starter` for the project.
- Use `AI assistant` only for the optional chat feature.
- Use `MCP endpoint` for the `/mcp` route.

## Style preferences

- Use active voice and second person.
- Keep sentences concise and direct.
- Use sentence case for headings.
- Lead with what a page or feature is for before the steps.
- Bold UI elements: Click **Deploy**.
- Code-format file names, commands, paths, and code references.
- Keep examples concrete and beginner-friendly.

## Content boundaries

- Prioritize generic documentation-site setup, customization, writing, and deployment.
- Do not document company-specific workflows.
- Do not invent Vercel pricing, quotas, or limits.
- Clearly label optional paid services, such as AI provider API usage.
- This site documents how to use and customize the starter, not how to administer a private product.
