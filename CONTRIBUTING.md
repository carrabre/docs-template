# Contributing

Thank you for improving this docs starter. Keep changes focused, generic, and useful for people launching their own documentation site.

## Local workflow

1. Install dependencies with `npm install`.
2. Start the site with `npm run dev`.
3. Make one focused change.
4. Run `npm run type-check`.
5. Run `npm run build`.
6. Open a pull request with a short summary and test notes.

## Writing standards

- Use active voice and second person.
- Keep sentences concise.
- Use sentence case for headings.
- Start each page with what the page is for.
- Bold UI labels, such as **Deploy**.
- Use code formatting for file names, paths, commands, and environment variables.
- Use concrete examples that a beginner can adapt.

## Template boundaries

- Keep copy platform-neutral unless a page is specifically about Next.js or Vercel deployment.
- Do not add company-specific language, pricing, quotas, or private workflows.
- Keep paid AI features optional and clearly labeled.
- Prefer simple MDX and shared components over page-specific styling.
